# Batch Reporting (Queue)

## Ziel

Viele Reports mit kurzer Bearbeitungszeit pro Fall.

## Ablauf

1. **Queue View**: Liste mit Prioritaeten.
2. **Quick Review**: Findings/Impression scannen.
3. **Freigabe**: Approval Dialog pro Fall.
4. **Next**: Naechster Report automatisch laden.

## UI Komponenten

- ReportQueue
- ImpressionPanel (Approve)
- Keyboard Shortcuts (Ctrl+Enter)
