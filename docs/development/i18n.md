# Internationalisierung (i18n)

Diese Dokumentation beschreibt das Internationalisierungssystem der MedGemma Radiology Anwendung.

## Übersicht

Die Anwendung verwendet [react-i18next](https://react.i18next.com/) für die Mehrsprachunterstützung. Aktuell werden folgende Sprachen unterstützt:

| Sprache | Code | Status |
|---------|------|--------|
| Deutsch | `de` | ✅ Vollständig |
| Englisch | `en` | ✅ Vollständig |

## Dateistruktur

```
src/i18n/
├── index.ts                    # i18n Konfiguration
└── locales/
    ├── de/                     # Deutsche Übersetzungen
    │   ├── common.json         # Allgemeine UI-Texte
    │   ├── report.json         # Befund-bezogene Texte
    │   ├── viewer.json         # DICOM Viewer Texte
    │   ├── batch.json          # Batch-Reporting Texte
    │   ├── settings.json       # Einstellungen
    │   └── errors.json         # Fehlermeldungen
    └── en/                     # Englische Übersetzungen
        ├── common.json
        ├── report.json
        ├── viewer.json
        ├── batch.json
        ├── settings.json
        └── errors.json
```

## Namespaces

Die Übersetzungen sind in logische Namespaces unterteilt:

| Namespace | Verwendung |
|-----------|------------|
| `common` | Navigation, Buttons, Status, allgemeine Labels |
| `report` | Befund-Editor, Impressions, QA, Templates |
| `viewer` | DICOM Viewer, Vergleichsmodus, Sync-Optionen |
| `batch` | Batch-Reporting Dashboard, Statistiken |
| `settings` | Benutzereinstellungen, Konfiguration |
| `errors` | Fehlermeldungen, Validierung |

## Für Entwickler

### Verwendung in Komponenten

```tsx
import { useTranslation } from 'react-i18next';

export function MyComponent() {
  const { t } = useTranslation('common'); // Namespace angeben
  
  return (
    <div>
      <h1>{t('navigation.home')}</h1>
      <button>{t('actions.save')}</button>
    </div>
  );
}
```

### Mehrere Namespaces

```tsx
const { t } = useTranslation(['report', 'common']);

// Namespace explizit angeben
t('report:findings.title')
t('common:actions.save')
```

### Interpolation (Variablen)

```tsx
// JSON: "wordCount": "{{count}} Wörter"
t('findings.wordCount', { count: 150 })

// JSON: "frame": "Bild {{current}} von {{total}}"
t('progress.frame', { current: 5, total: 20 })
```

### Pluralisierung

```tsx
// JSON:
// "study": "{{count}} Studie",
// "study_plural": "{{count}} Studien"
t('study', { count: 1 })  // → "1 Studie"
t('study', { count: 5 })  // → "5 Studien"
```

### Dynamische Schlüssel

```tsx
const status = 'approved'; // pending | inProgress | draft | approved
t(`status.${status}`)      // → "Freigegeben"
```

### Außerhalb von React (Class Components, Utilities)

```tsx
import i18n from '@/i18n';

// Direkte Verwendung
const label = i18n.t('common:actions.save');

// Sprache wechseln
i18n.changeLanguage('en');
```

### Neue Übersetzungsschlüssel hinzufügen

1. **Schlüssel in allen Sprachdateien hinzufügen:**

```json
// src/i18n/locales/de/common.json
{
  "myFeature": {
    "title": "Mein Feature",
    "description": "Beschreibung hier"
  }
}

// src/i18n/locales/en/common.json
{
  "myFeature": {
    "title": "My Feature",
    "description": "Description here"
  }
}
```

2. **In Komponente verwenden:**

```tsx
const { t } = useTranslation('common');
return <h1>{t('myFeature.title')}</h1>;
```

### Best Practices für Entwickler

- ✅ **Immer alle Sprachen aktualisieren** – Nie nur eine Sprachdatei ändern
- ✅ **Semantische Schlüssel verwenden** – `patient.age` statt `label1`
- ✅ **Gruppierung nutzen** – Zusammengehörige Texte in Objekten
- ✅ **Interpolation für dynamische Werte** – Keine String-Konkatenation
- ✅ **Namespace passend wählen** – Feature-spezifische Texte richtig einordnen
- ❌ **Keine hartcodierten Strings** – Alle UI-Texte müssen übersetzt sein
- ❌ **Keine HTML in Übersetzungen** – Markup in Komponenten halten

## Für Übersetzer

### JSON-Format

Übersetzungsdateien sind im JSON-Format:

```json
{
  "sectionName": {
    "key": "Übersetzter Text",
    "keyWithVariable": "Hallo {{name}}!",
    "nestedSection": {
      "deepKey": "Tief verschachtelter Text"
    }
  }
}
```

### Wichtige Regeln

1. **Schlüssel nicht ändern** – Nur die Werte (rechte Seite) übersetzen
2. **Variablen beibehalten** – `{{variable}}` nicht übersetzen oder entfernen
3. **Struktur beibehalten** – Verschachtelung und Kommas beachten
4. **Kontext beachten** – Medizinische Fachbegriffe korrekt verwenden

### Beispiel-Übersetzung

```json
// Original (Deutsch)
{
  "findings": {
    "title": "Befund",
    "wordCount": "{{count}} Wörter"
  }
}

// Übersetzung (Englisch)
{
  "findings": {
    "title": "Findings",
    "wordCount": "{{count}} words"
  }
}
```

### Medizinische Terminologie

| Deutsch | Englisch | Kontext |
|---------|----------|---------|
| Befund | Findings | Radiologischer Befundtext |
| Beurteilung | Impression | Zusammenfassung/Diagnose |
| Voruntersuchung | Prior Study | Frühere Bildgebung |
| Modalität | Modality | CT, MRT, Röntgen, etc. |
| Freigabe | Approval | Befundautorisierung |
| Diktat | Dictation | Spracherkennung |

### Validierung

Vor dem Einreichen prüfen:

- [ ] JSON-Syntax gültig (keine fehlenden Kommas/Klammern)
- [ ] Alle Variablen `{{...}}` vorhanden
- [ ] Keine leeren Werte
- [ ] Fachbegriffe korrekt verwendet
- [ ] Konsistente Terminologie innerhalb der Datei

## Neue Sprache hinzufügen

### 1. Ordner und Dateien erstellen

```bash
mkdir src/i18n/locales/fr
cp src/i18n/locales/en/*.json src/i18n/locales/fr/
```

### 2. Dateien übersetzen

Alle kopierten JSON-Dateien ins Französische übersetzen.

### 3. In i18n/index.ts registrieren

```typescript
// Imports hinzufügen
import frCommon from './locales/fr/common.json';
import frReport from './locales/fr/report.json';
// ... weitere Imports

// In resources ergänzen
export const resources = {
  de: { /* ... */ },
  en: { /* ... */ },
  fr: {
    common: frCommon,
    report: frReport,
    viewer: frViewer,
    batch: frBatch,
    settings: frSettings,
    errors: frErrors,
  },
} as const;
```

### 4. Sprachauswahl in Settings erweitern

In `src/pages/Settings.tsx` die neue Sprache zur Auswahl hinzufügen.

## Sprachauswahl

Die Sprache wird in folgender Reihenfolge bestimmt:

1. **Benutzereinstellung** – Gespeichert in localStorage (`medgemma-user-preferences`)
2. **Browser-Sprache** – Automatische Erkennung
3. **Fallback** – Deutsch (`de`)

### Programmatisch wechseln

```typescript
import i18n from '@/i18n';
import { useUserPreferences } from '@/hooks/useUserPreferences';

// In Komponente
const { updatePreferences } = useUserPreferences();

const handleLanguageChange = (lang: 'de' | 'en') => {
  i18n.changeLanguage(lang);
  updatePreferences({ uiLanguage: lang });
};
```
