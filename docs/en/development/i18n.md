# Internationalization (i18n)

This documentation describes the internationalization system of the Radiolyze application.

## Overview

The application uses [react-i18next](https://react.i18next.com/) for multi-language support. The following languages are currently supported:

| Language | Code | Status |
|----------|------|--------|
| German | `de` | ✅ Complete |
| English | `en` | ✅ Complete |

## File Structure

```
src/i18n/
├── index.ts                    # i18n configuration
└── locales/
    ├── de/                     # German translations
    │   ├── common.json         # General UI text
    │   ├── report.json         # Report-related text
    │   ├── viewer.json         # DICOM Viewer text
    │   ├── batch.json          # Batch reporting text
    │   ├── settings.json       # Settings
    │   └── errors.json         # Error messages
    └── en/                     # English translations
        ├── common.json
        ├── report.json
        ├── viewer.json
        ├── batch.json
        ├── settings.json
        └── errors.json
```

## Namespaces

Translations are organized into logical namespaces:

| Namespace | Usage |
|-----------|-------|
| `common` | Navigation, buttons, status, general labels |
| `report` | Report editor, impressions, QA, templates |
| `viewer` | DICOM Viewer, comparison mode, sync options |
| `batch` | Batch reporting dashboard, statistics |
| `settings` | User preferences, configuration |
| `errors` | Error messages, validation |

## For Developers

### Usage in Components

```tsx
import { useTranslation } from 'react-i18next';

export function MyComponent() {
  const { t } = useTranslation('common'); // specify namespace
  
  return (
    <div>
      <h1>{t('navigation.home')}</h1>
      <button>{t('actions.save')}</button>
    </div>
  );
}
```

### Multiple Namespaces

```tsx
const { t } = useTranslation(['report', 'common']);

// specify namespace explicitly
t('report:findings.title')
t('common:actions.save')
```

### Interpolation (Variables)

```tsx
// JSON: "wordCount": "{{count}} Wörter"
t('findings.wordCount', { count: 150 })

// JSON: "frame": "Bild {{current}} von {{total}}"
t('progress.frame', { current: 5, total: 20 })
```

### Pluralization

```tsx
// JSON:
// "study": "{{count}} Studie",
// "study_plural": "{{count}} Studien"
t('study', { count: 1 })  // → "1 Studie"
t('study', { count: 5 })  // → "5 Studien"
```

### Dynamic Keys

```tsx
const status = 'approved'; // pending | inProgress | draft | approved
t(`status.${status}`)      // → "Freigegeben"
```

### Outside of React (Class Components, Utilities)

```tsx
import i18n from '@/i18n';

// Direct usage
const label = i18n.t('common:actions.save');

// Change language
i18n.changeLanguage('en');
```

### Adding New Translation Keys

1. **Add the key to all language files:**

```json
// src/i18n/locales/de/common.json
{
  "myFeature": {
    "title": "Mein Feature",
    "description": "Beschreibung hier"
  }
}

// src/i18n/locales/en/common.json
{
  "myFeature": {
    "title": "My Feature",
    "description": "Description here"
  }
}
```

2. **Use in a component:**

```tsx
const { t } = useTranslation('common');
return <h1>{t('myFeature.title')}</h1>;
```

### Best Practices for Developers

- ✅ **Always update all languages** – Never change only one language file
- ✅ **Use semantic keys** – `patient.age` instead of `label1`
- ✅ **Use grouping** – Related texts in objects
- ✅ **Interpolation for dynamic values** – No string concatenation
- ✅ **Choose the appropriate namespace** – Correctly categorize feature-specific text
- ❌ **No hardcoded strings** – All UI text must be translatable
- ❌ **No HTML in translations** – Keep markup in components

## For Translators

### JSON Format

Translation files use JSON format:

```json
{
  "sectionName": {
    "key": "Translated text",
    "keyWithVariable": "Hello {{name}}!",
    "nestedSection": {
      "deepKey": "Deeply nested text"
    }
  }
}
```

### Important Rules

1. **Do not change keys** – Only translate the values (right-hand side)
2. **Preserve variables** – Do not translate or remove `{{variable}}`
3. **Preserve structure** – Respect nesting and commas
4. **Respect context** – Use correct medical terminology

### Example Translation

```json
// Original (German)
{
  "findings": {
    "title": "Befund",
    "wordCount": "{{count}} Wörter"
  }
}

// Translation (English)
{
  "findings": {
    "title": "Findings",
    "wordCount": "{{count}} words"
  }
}
```

### Medical Terminology

| German | English | Context |
|--------|---------|---------|
| Befund | Findings | Radiology report text |
| Beurteilung | Impression | Summary/diagnosis |
| Voruntersuchung | Prior Study | Previous imaging |
| Modalität | Modality | CT, MRI, X-ray, etc. |
| Freigabe | Approval | Report authorization |
| Diktat | Dictation | Speech recognition |

### Validation

Check before submitting:

- [ ] JSON syntax valid (no missing commas/brackets)
- [ ] All variables `{{...}}` present
- [ ] No empty values
- [ ] Technical terms used correctly
- [ ] Consistent terminology within the file

## Adding a New Language

### 1. Create folder and files

```bash
mkdir src/i18n/locales/fr
cp src/i18n/locales/en/*.json src/i18n/locales/fr/
```

### 2. Translate the files

Translate all copied JSON files into French.

### 3. Register in i18n/index.ts

```typescript
// Add imports
import frCommon from './locales/fr/common.json';
import frReport from './locales/fr/report.json';
// ... additional imports

// Add to resources
export const resources = {
  de: { /* ... */ },
  en: { /* ... */ },
  fr: {
    common: frCommon,
    report: frReport,
    viewer: frViewer,
    batch: frBatch,
    settings: frSettings,
    errors: frErrors,
  },
} as const;
```

### 4. Extend the language selector in Settings

Add the new language to the selection in `src/pages/Settings.tsx`.

## Language Selection

The language is determined in the following order:

1. **User preference** – Stored in localStorage (`medgemma-user-preferences`)
2. **Browser language** – Automatic detection
3. **Fallback** – German (`de`)

### Changing programmatically

```typescript
import i18n from '@/i18n';
import { useUserPreferences } from '@/hooks/useUserPreferences';

// In component
const { updatePreferences } = useUserPreferences();

const handleLanguageChange = (lang: 'de' | 'en') => {
  i18n.changeLanguage(lang);
  updatePreferences({ uiLanguage: lang });
};
```
