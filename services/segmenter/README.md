# Radiolyze Segmenter

Standalone HTTP microservice that produces tissue-segmentation masks and 3D
meshes (GLB + VTP) from a CT series fetched via DICOMweb.

## Endpoints

| Route | Method | Purpose |
|---|---|---|
| `/health` | GET | Liveness + reports `gpu` flag |
| `/segment/bone` | POST | HU-thresholding bone preset (M1, no ML) |
| `/segment/total` | POST | TotalSegmentator multi-organ |
| `/jobs/{id}` | GET | Status + manifest |
| `/jobs/{id}/manifest` | GET | Raw `manifest.json` |
| `/jobs/{id}/mesh/{label_id}?format=glb\|vtp` | GET | Binary mesh stream |
| `/jobs/{id}/mask/{label_id}` | GET | NIfTI mask stream |
| `/jobs/{id}/dicom-seg` | GET | Multi-class DICOM SEG (when M4 enabled) |

The `job_id` is generated by the backend orchestrator and passed in the request
body so the orchestrator owns identity.

## Storage

Outputs are written to `SEGMENTATION_DATA_DIR` (default `/data/segmentations`).
The orchestrator mounts the same volume and streams meshes directly from disk
for performance. Layout:

```
$SEGMENTATION_DATA_DIR/<job_id>/
  raw/ct.nii.gz
  masks/<id>_<name>.nii.gz
  meshes/<id>.glb
  meshes/<id>.vtp
  manifest.json
```

Manifests are written atomically (`.tmp` → `rename`) so readers only ever see a
complete file when the status flips to `done`.

## Config

| Env | Default | Purpose |
|---|---|---|
| `SEGMENTATION_DATA_DIR` | `/data/segmentations` | Output root |
| `DICOM_WEB_BASE_URL` | `http://orthanc:8042/dicom-web` | DICOMweb root |
| `DICOM_WEB_USERNAME` / `DICOM_WEB_PASSWORD` | — | Basic auth |
| `SEGMENTER_MAX_PARALLEL_WADO` | `8` | Concurrent WADO instance fetches |
| `BONE_HU_THRESHOLD` | `300` | HU cutoff for bone preset |
